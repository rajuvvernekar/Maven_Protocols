## Common Aliases in Feedback

These terms in feedback map to specific tools:

- "Payin" → `cashier_payins`
- "Withdrawal" → `withdrawal_request`
- "IPO" → `ipo_application`
- "Kite" (orders context) → `kite_orders` or `kite_order_history`
- "Kite Gtt Orders" → `kite_gtt`
- "Ledger Balance" / "Ledger Report" → `ledger_report`
- "MF" / "Mutual Fund" / "Mutual Funds" / "Coin" → check context, usually `mf_order_history`
- "MF holdings" / "MF holdings discrepancy" → `console_mf_holdings` or `console_mf_pseudo_holdings`
- "MF Redemption" → `mf_order_history`
- "Mf Discrepancy" / "MF discrepancy" → `console_mf_pseudo_holdings`
- "Mf Failed Payment" → `mf_order_history` + `fund_allocation_report`
- "MF Fund allocation report" → `fund_allocation_report`
- "Mf Mandate Debit" → `mandate_debit_report`
- "Mf Order History" / "MF Order history" / "Mf Order" → `mf_order_history`
- "Mf Pseudo Holdings" → `console_mf_pseudo_holdings`
- "Mf Tradebook" → `console_mf_tradebook`
- "Mutual Fund external trades" → `console_mf_external_trades`
- "Console Mandates" → `mandate_report`
- "Qs Payouts" / "QS" / "Qs payout" → `crux_qs_payouts`
- "SIP" → `sip_report`
- "Account Debit" → check context (could be `ledger_report`, `cashier_payins`, or `mf_order_history`)
- "Segment Activation" / "Activating F&o" / "Segment addition" → `account_modification_report` or `pan_status`
- "Account Modification" → `account_modification_report`
- "Response Quality" → not a tool issue, skip
- "Portfolio Analysis" → not a tool issue, skip
